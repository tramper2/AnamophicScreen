## 1. 개요 (Overview)
- **프로젝트 명:** WebGL 기반 애나모픽 디스플레이 시뮬레이터 (Anamorphic Web Previewer)
- **목적:** 사용자가 업로드한 동영상이 다양한 애나모픽 디스플레이 환경(L자, 원통, 구체 등)에서 어떻게 왜곡되고 시각화되는지 브라우저에서 실시간으로 프리뷰할 수 있는 정적 웹 애플리케이션 개발.
- **배포 환경:** GitHub Pages (클라이언트 사이드 연산 100%)

## 2. 사용자 스토리 및 핵심 기능 (User Stories & Core Features)
- **비디오 소스 로드:** 사용자는 로컬 비디오 파일(MP4, WebM 등)을 브라우저에 업로드(메모리 참조)하거나 기본 샘플 비디오를 선택하여 테스트할 수 있어야 함.
- **디스플레이 프리뷰 모드 전환:** 사용자는 실시간으로 스크린 형태를 변경할 수 있어야 함.
  - 모드 1: 평면 (기준점)
  - 모드 2: L자형 스크린 (정면 + 바닥/측면 90도 절곡)
  - 모드 3: 원통형 스크린 (Cylinder 내부 매핑)
  - 모드 4: 스피어/돔 스크린 (Sphere 내부 매핑)
- **애나모픽 착시(Sweet Spot) 시뮬레이션:**
  - 특정 고정 시점(가상 프로젝터 시점)에서 바라볼 때만 영상이 왜곡 없이 올바른 비율로 보이도록 투사하는 기능.
  - 사용자가 마우스 드래그로 카메라를 움직이면 왜곡이 발생하고, '정점 보기(Reset View)' 버튼을 누르면 가상 착시 완벽 시점으로 카메라가 부드럽게 이동.
- **실시간 파라미터 조절:** 화면의 곡률, 꺾임 각도, 스크린 크기 등을 조절할 수 있는 컨트롤 패널 제공.

## 3. 비기능적 요구사항 (Non-Functional Requirements)
- **서버리스 아키텍처:** GitHub Pages에 배포되므로 모든 영상 처리 및 3D 렌더링은 사용자 브라우저(WebGL)에서 구동되어야 함. 서버 백엔드가 없어야 함.
- **성능:** 1080p 60fps 비디오 입력 시에도 최소 45fps 이상의 3D 렌더링 프레임레이트 유지.
- **반응형 웹 UI:** 컨트롤러 영역과 3D 뷰포트 영역이 다양한 스크린 해상도에서 깨지지 않고 배치되어야 함.
